## 启动相机
新建一个命令行，输入以下命令：
```
source ~/deploy/tmp/OrbbecSDK_develop/orbbec_ws/devel/setup.bash
roslaunch orbbec_camera multi_camera.launch
```
如果不报错，说明启动成功（黄色字的警告可以忽略）

## 启动机械臂
⚠️⚠️⚠️通电时：先给从臂通电，再给主臂通电（先按插线板开关，再插主臂电源插头）

⚠️⚠️⚠️断电时：先给主臂断电，再给从臂断电（先拔主臂电源插头，再按插线板开关）

- 确认第三人称视角相机的三脚架位置与地上的标志对齐（正常情况就是对齐的，只要没有刻意挪动过一般不会有问题）
- 确认两机械臂均在正常的初始位置：各关节点位对齐，主从臂方向一致
- 确认面包板上处于主从臂连接状态，且各杜邦线未松动（如果杜邦线脱落，不知道如何接线，请速联系寒哥）
- 插电：⚠️⚠️先给从臂通电，再给主臂通电⚠️⚠️（先按插线板开关，再插主臂电源插头）
- 等待约20秒，小幅度移动主臂，检查二者是否已经同步，如果没同步就接着等待（若超过1分钟仍未同步，可以重新断电再通电）
- 在之后的数据收集流程中，都应操作主臂来控制从臂，不要直接摆弄从臂

## 启动数据采集软件（详情可见[数据采集软件](../DataCollection/数据收集软件说明/数据采集软件.md)）
通过以下指令可以快速启动数据收集软件：
```bash
cd /home/pc3/data_collect/songlings
conda activate piper_clt
python data_collect.py
```
打开软件后，先在左下方输入框中输入本次采集的所有数据保存的文件夹，然后点击右边的确定。（这里填的文件夹是相对于data/tmp的路径）

## 采集数据
- 操纵主臂到初始位置，保证屏幕上的初始位置鉴定程序在XZY上全绿
- 确保机械臂夹爪闭合
- 确保腕上相机视角面向桌面
- 点击数据采集软件的开始采集
- 操纵机械臂完成动作
- 点击数据采集软件的停止采集
- 完成所有数据的采集之后，操控机械臂让主臂和从臂均水平放置稳妥。遵循⚠️⚠️先主后从⚠️⚠️的顺序给两机械臂断电。

## 数据采集注意事项
1. 动作可以不近乎最优，可以有一些错误动作（甚至故意造一些错误数据），但这种错误动作不能是细粒度的（比如剧烈抖动），得是粗粒度的（比如place错位置重新抓取），错误动作后需要立即修正。有失误时得在对应存储的文件名处用中文标记一下
2. 初始位置抬在空中某限定范围内
3. 物体每次随意摆放，但要在一个大的限定范围内。可以偶尔制造一些混乱场景，比如靠的非常近，或者出现堆叠现象
4. 动作不要太快，慢慢的，采集的速度所有人尽量保持一致
5. 每个人刚开始采集时，采满5条需要找罗旭或王寒check数据质量

## 操作注意事项
1. 操纵机械臂时推荐使用两只手，右手抓住前端，左手扶住臂身
2. 操纵机械臂时，解耦的动作也许能够更便于模型学习。即每次动作只让一个或两个关节移动。
3. 建议在采集过程中，两个动作间加入一些适当的停顿，便于后续数据标注同学分解各个动作。
4. 主从臂方式采集数据时，务必遵守“通电时先从后主，断电时先主后从”的顺序。

## 故障排除
主从臂方式采集数据时，已知的故障情况有两种：
1. 操纵主臂的过程中，从臂的底部关节突然快速转回初始位置，此后主臂仍然可以继续控制从臂。
   遇到该故障，应先操控主臂将两条机械臂都放平；然后遵循⚠️⚠️先主后从⚠️⚠️的顺序给两机械臂断电；将两机械臂都放置到初始位置之后，重新按照⚠️⚠️先从后主⚠️⚠️的顺序通电。
2. 操纵主臂的过程中，从臂停滞不受控或动作卡顿，此时从臂无法正常与主臂同步。
   遇到该故障，应停止操控主臂，尽量保持两臂姿态相似；然后呼叫其他同学来帮忙扶住两个机械臂，遵循⚠️⚠️先主后从⚠️⚠️的顺序给两机械臂断电；将两机械臂都放置到初始位置之后，重新按照⚠️⚠️先从后主⚠️⚠️的顺序通电。
