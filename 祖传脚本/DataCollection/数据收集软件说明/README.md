# 背景说明

数据收集软件旨在通过编写脚本/软件为机械臂数据收集提供辅助，这些软件通过**采集**数据，并将数据**转化**为能够被轻易用于模型训练的格式，来推动项目的持续前进。

正如上述加重的字体，本部分主要包含两个不同类型的程序，即[**数据采集程序**](./数据采集软件.md)和[**数据格式转化程序**](./数据格式转化.md)。

*接下来关于程序内容的描述是建立在读者拥有基本的计算机科学与工程知识的假设之上的，如果有感到疑惑的地方，可先行询问各类AI模型，或询问mizuki-p\@foxmail.com*

---

数据采集程序的对象是松灵制造的机械臂，采集程序的输入是机械臂运动时的各种信息和摄像头拍摄到的图像，采集程序的输出是一个包含上述各种信息的文件夹。

数据格式转化程序的输入是采集程序的输出，而输出则是lerobot格式的数据集。

---

更加具体的来说，此处的松灵制造的机械臂是**松灵Piper**机械臂，该机械臂包含六个关节和一个夹爪或者一个示教器。（建议通过实际操作来建立感性认知）

我们可以通过两种方式来描述机械臂的某一种状态（这里的状态是指机械臂在空间中的位置，与夹爪无关），即EEF（End of Postition）方式和Joint方式：
* EEF方式：EEF通过描述机械臂末端在空间中的位置来描述机械臂所处的状态，具体而言，EEF是一个向量(X, Y, Z, RX, RY, RZ)，其中X, Y, Z是符合人类正常直觉的三维笛卡尔坐标系，RX,RY,RZ则是末端的欧拉角，用于表示末端的旋转状态。根据我的实践经验，RX,RY,RZ对指导我们编写程序/探寻运动轨迹等基本没啥用，但是模型还是必须要学这个的。
* Joint方式：Joint方式主要通过描述六个关节的旋转角度来表述机械臂在空间中的状态

EEF方式的六个值都是Int类型，其中XYZ坐标的单位为0.001 mm。
Joint方式的六个值都是Int类型，每个值的单位为0.001度。

夹爪和示教器可以被视作同一个东西，区别在于示教器更好操作（毕竟是为了更好得操作来设计出来的）。其状态通过开合角度、力度来定义，在数据数据、模型训练和推理时，力度这个参数都是被忽略的（模型推理时，力度是被写死的）。开合角度是Int类型，单位不清楚。

在机械臂上的数据采集结果以机械臂运动过程中产生的EEF、Joint、Gripper状态序列的形式展示，序列以时间先后的顺序排列。

在摄像头上的数据采集结果以摄像机拍摄到的RGB图像和深度图像序列的像是展示，图像以png格式保存， 文件名为时间戳。

---

在数据采集完成后，需要对数据进行标注并将数据组织成lerobot数据集的格式。

幸运的是，数据格式转化过程可以通过复用[openpi](https://github.com/Physical-Intelligence/openpi)仓库的代码完成，我们需要完成的是上述数据的一些预处理。

预处理的主要内容包括三个：
* 数据同步与下采样
* 零动作检测与处理
* 异动检测与处理

通过以上三步，可以合法预处理数据，然后可以方便地构建正确的数据集。
