# 数据标注软件

> 软件都保存在/home/pc3/data_collect/data_anno下

数据标注的目的是为VLA模型提供文本输入，为VLM提供图像和文本对，并裁剪出每条数据中重要的部分。

数据标注软件的输入是数据采集的结果，即一条一条的数据，通过与用户的交互，其输出为标注结果。

标注结果体现为一个JSON文件，该文件具有以下结构：
```json5
{
  // 任务名称，通过"所属任务/采集时间"命名，其实所属任务和采集时间都是文件夹名字
  "task_name": "re_sweep/04-10AAA20:17:43",
  // 标注者，可以在这里写下你的大名，用于计算KPI（其实并没有）
  "annotator": "wade",
  // 当前状态，可选状态有[unfinished, procedding, finished]
  // unfinished表示未开始（尽管这个词是表示未结束，但是这已经是一个历史错误，积重难返了
  // procedding表示正在标注中
  // finished表示已经标完了
  "status": "finished",
  // 同步后的图像文件列表
  "frames": [
		// 每个元素都有相同的格式
		// 第一个是front_color图像
		// 第二个是wrist_color图像
		// 第三个是front_depth图像
		// 第四个是wrist_depth图像
		// 第五个是四张图像的平均时间戳
	    [
	      "1744287463_996841000.png",
	      "1744287463_994838000.png",
	      "1744287464_001546000.png",
	      "1744287463_982255200.png",
	      1744287463.99387
	    ],
    ]
  // 标注数据列表
  "spans": [
		// 每个元素都是一个对象
		{
		  // 这条标注起始的位置，对应frames列表中的第start_index帧
		  "start_index": 50,
		  // 这条标注结束的位置，对应frames列表中的第end_index帧
		  "end_index": 471,
		  // 文本标注
		  "annotation": "pick up the rag",
		  // 该条标注是否用于VLM训练
		  "used_for_vlm": true
		},
  ]
}
```

JSON文件放置于对应的每条数据的文件夹下，文件名统一为ano.json。

---

可以通过以下指令启动标注软件
```bash
cd /home/pc3/data_collect/data_anno
conda activate piper_clt
python data_anno.py
```

然后在局域网内访问对应地址和对应端口的index.html文件即可。

数据标注软件工作的对象是一个文件夹，位于/home/pc3/data_collect/data_anno/data/ano下，其结构和[数据采集软件](./数据采集软件.md)的输出一样，只是多了个ano.json文件。

> ⚠️ 需要手动将采集的数据移动到上述目录中，该软件才能读取到对应的数据。

![数据标注截图](./imgs/数据标注.png)

数据标注的网页界面如上所示，由于各种各样的原因，请手动访问 **/index.html** 路径来进入该页面。

页面的设计大体上符合直觉，左侧是需要标注的数据，右侧是具体的操作区域。因为是时序数据，所以有一个进度条来表示当前的位置。进度条可以点击来调整当前位置。

两张图像分别是当前时刻的front和wrist图像，下方有五个按钮，播放可以直接一帧一帧按顺序播放图像，后面可以精准调整当前位置，横条可以调整播放速度。

点击下方**设为起始**可以开始一条span，点击**设为结束**可以结束一条span，结束后，在右面输入框输入标注和选择是否用于VLM训练后，点击保存标注即可保存当前的标注。**取消选择**和**删除选中**的功能和名称一样。标注好的数据位于下方表格中，表格中的元素也可被选中，其效果与在进度条中选中对应的span是一致的。

下方有三个radio选项，用于表示该条数据的状态。剩下两个可交互组件的含义与其名称是一致的，这里就不赘述了。

左侧下方有两个按钮，重新请求会更新已有的数据，不会改变数据的条数，右边会从服务器上重新读取数据文件夹，有可能会改变数据的条数。
