# 数据格式转化

## 软件说明


数据格式转化主要通过/home/pc3/data_collect/openpi/convert_data_new.py文件实现

其可以通过以下方式调用

```bash
cd /home/pc3/data_collect/openpi/
source .venv/bin/activate
python convert_data_new.py \
	--root_path temp
	--save_path tmp/dataset
	--continue_on False
```

其中root_path是数据来源，其中存放一条一条的轨迹数据，save_path是输出路径，continue_on表示是否在之前的数据集上添加新数据。

当continue_on为False时，该脚本从root_path中读取数据，并创建save_path指向的文件夹，如果文件夹已经存在的话，脚本是会报错的~~*（这是lerobot库本身的行为，尽管我认为这种行为相当弱智）*~~，然后将构建好的数据集放在save_path下。
当continue_on为True时，该脚本先从save_path恢复之前存储的数据集，然后从root_path读取新数据并添加到数据集中，然后保存到save_path中。

数据集构建好后，就可以打包（也可以不打包）传到服务器上进行训练了。

## 流程说明

一般来说，一次构建通常会涉及到多次采集的数据，而这些数据通常在文件系统上以以下方式进行存储

```
data/ano
|- 第一次采集
|---traj1_04_01_24_00_00
|---traj2_04_01_24_00_01
|- 第二次采集
|---traj1_04_01_1_00_00
|---traj2_04_01_1_02_00
```

而root_path指向的目录应当只包含"traj1_04_01_24_00_00"等目录，而不要"第一次采集"这一层级的目录，从而需要将这些具体的数据目录组织起来。

在具体实践中，我们通过脚本**agg_data_link.sh**脚本实现（跟上面那个脚本在同一个目录下），该脚本通过将需求的目录的**软连接**放到一个目录中来组织目录结构。该脚本的使用方法如下：

```bash
conda activate piper_clt
python agg_data_link.sh <target_dir> [source_dirs..]
```

该脚本有一个必需的参数target_dir，这个会用作数据格式转化脚本的root_path，和一个可变长度参数source_dirs，顾名思义，我就不多做解释了。如果是上面那个需求，那么这里的source_dirs就是"data/ano/第一次采集 data/ano/第二次采集"
